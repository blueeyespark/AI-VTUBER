# Blue Creation and Learning Plan

## Reference-aware art and model creation

Blue should not throw every reference into one giant prompt. The new prototype creates a reference manifest first:

- identity reference
- outfit reference
- pose/motion reference
- color palette
- expression sheet
- background/world reference
- negative reference / avoid list
- weight and provenance for each file

This is the foundation for better image generation, texture generation, 3D model edits, Live2D model prep, and repeatable outputs.

## 3D / Live2D production

Blue's asset pipeline should separate source files from candidates and installed builds:

1. Archive original VRM, Blender, texture, and reference files.
2. Create reversible repair tasks.
3. Export versioned candidates.
4. Run smoke tests.
5. Install only the approved candidate.

## Teaching mode

When you ask Blue to teach drawing, coding, streaming, modeling, or any other topic, Blue should:

1. Research current sources.
2. Explain the topic in your skill level.
3. Give one small practice task.
4. Give one stretch challenge.
5. Check your understanding.
6. Save progress only with approval.
